## 前提

- `starter-kit/BookProject` は受講者配布用で、トップページとAPI起動確認のみ実装済みと判断した。
- `basic_answer` は基本機能の解答例と判断した。
- `additional_answer` は追加課題まで含めた解答例と判断した。
- `docs/images/01_home.png` から `09_book_register_error.png` は基本機能の画面例、`10_login.png` から `16_user_restrictions.png` は追加機能込みの画面例と判断した。

## 結論

基本機能については、仕様書、CSSクラス対応表、`starter-kit`、`basic_answer`、画面画像の整合性はおおむね取れている。

APIのURL、DB初期データ、基本画面の表示項目、検索条件、エラーメッセージは、実装と仕様書で大きな矛盾は見つからなかった。

一方で、追加課題まで含めて読む場合は、CSSクラス対応表の不足と、一部画像の文脈不足がある。受講者が資料単体で読む場合に混乱しやすいため、下記の修正または補足を推奨する。

## 指摘事項

### 1. CSSクラス対応表に追加課題用classが載っていない

重要度: 要修正

`additional_answer/css/style.css` では、追加課題用に次のclassが定義・使用されている。

- `login-area`
- `login-user`
- `home-info-list`
- `home-info-item`
- `home-info-value`
- `recent-book-list`
- `sort-area`
- `success-message`
- `preview-image`
- `admin-menu`

しかし、`docs/CSSクラス対応表.md` にはこれらが掲載されていない。

CSS対応表を「スターターキット・基本機能用」として扱うなら、表の冒頭にその範囲を明記するとよい。総合演習ガイド全体の対応表として扱うなら、追加課題用classも追記する必要がある。

### 2. 画像プレビュー画像に入力エラーが同時に表示されている

重要度: 要確認

`docs/総合演習ガイド.md` の「10.2 画像プレビュー」では、画像パス入力時にプレビューが表示されることを説明している。

対応画像 `docs/images/13_image_preview.png` では、画像プレビュー自体は表示されているが、同時に赤字で「書籍名を入力してください。」という入力エラーも表示されている。

画像プレビューの説明だけを見る受講者には、プレビュー機能と入力チェック機能が混ざって見える。画像プレビューの例として使うなら、エラーメッセージが出ていない状態の画像へ差し替えることを推奨する。

### 3. 削除Modalの文言が仕様書と実装・画像で少し違う

重要度: 軽微

`docs/総合演習ガイド.md` のUC006表示項目では、確認メッセージを「`{書籍名} を削除してもよろしいですか？`」としている。

一方、`basic_answer/js/components/book-delete-modal.js` と `additional_answer/js/components/book-delete-modal.js`、および `docs/images/07_book_delete_modal.png` では、本文が「次の書籍を削除します。よろしいですか。」で、書籍名は次の行に太字で表示される。

意味は同じだが、教材としては文言をどちらかに寄せた方がよい。

### 4. 追加課題の画像は「全追加機能を同時に入れた状態」であることを補足した方がよい

重要度: 補足推奨

`docs/images/11_top_info.png`、`12_sort_list.png`、`13_image_preview.png`、`15_success_message.png` などは、ヘッダーにログイン中ユーザーとログアウトボタンが表示されている。

これは `additional_answer` がログイン機能込みの完成状態だから自然だが、ガイド上では「10.1 並び替え」「10.2 画像プレビュー」「10.5 トップページ情報表示」がログイン機能より前に説明されている。

追加課題の画像セクションに「以降の画面例は、追加課題をすべて適用した解答例の画面です」と補足すると、単体課題の仕様と画像の見た目の差を誤解されにくい。

## 整合性確認済みの事項

### 仕様書とstarter-kit

- `starter-kit/BookProject/index.html` は、ヘッダー、サイドバー、モバイルメニュー、`main#app`、`modal-area`、Bootstrap Iconsを含んでおり、提供物の説明と一致している。
- `starter-kit/BookProject/js/app.js` はトップページのみを表示し、未実装URLでは未実装メッセージを出す。ガイドの「総合演習の中で追加する」という説明と一致している。
- `starter-kit/BookProject/api/server.js` は `/api/test` のみを実装しており、スターターキットとしての説明と一致している。
- `starter-kit/BookProject/db/books.sql` は `books` と追加課題用の `users` を含んでおり、ガイドのデータベース仕様と一致している。

### 仕様書とbasic_answer

- `/`, `/index.html`, `/books`, `/books/new`, `/books/{id}` のルーティングは仕様書と一致している。
- `GET /api/v1/books`, `GET /api/v1/books?keyword=xxx`, `GET /api/v1/books/:id`, `POST /api/v1/books`, `PUT /api/v1/books/:id`, `DELETE /api/v1/books/:id` は、仕様書のAPI一覧と一致している。
- `keyword=go` の検索結果は `Django開発` と `Go言語` で、画面説明と一致している。
- 登録・更新の必須チェックは、書籍名、著者名、価格を対象としており、仕様書と一致している。
- 登録成功時は `alert('書籍を登録しました。')` の後に一覧へ移動する。基本機能の仕様と一致している。

### 仕様書とadditional_answer

- `POST /api/v1/login` は `admin/admin123` と `user/user123` を認証でき、ガイドのログイン仕様と一致している。
- `GET /api/v1/books/summary` は、`count: 10` とID降順の最新3件を返し、ガイドのレスポンス例と一致している。
- `GET /api/v1/books?sort=price_asc` と `sort=price_desc` は、価格の昇順・降順で返り、並び替え仕様と一致している。
- `role=user` で登録APIを呼ぶと403になり、権限管理の方針と一致している。

### 画像ファイル

- `docs/総合演習ガイド.md` で参照されている画像16件は、すべて `docs/images/` に存在する。
- 基本機能の画像 `01_home.png` から `09_book_register_error.png` は、基本機能の説明とおおむね一致している。
- 追加機能の画像 `10_login.png` から `16_user_restrictions.png` は、`additional_answer` の実装内容とおおむね一致している。

## 実行確認

`basic_answer/api` で確認した内容:

- `GET /api/test` が `{"status":"ok","message":"APIサーバー稼働中！"}` を返す。
- `GET /api/v1/books` が10件を返す。
- `GET /api/v1/books?keyword=go` が `Django開発` と `Go言語` を返す。
- `GET /api/v1/books/1` が `Pythonの教科書` を返す。
- 必須項目不足の `POST /api/v1/books` が400を返す。
- 存在しない `GET /api/v1/books/99999` が404を返す。

`additional_answer/api` で確認した内容:

- `POST /api/v1/login` が `admin/admin123` と `user/user123` を認証できる。
- `GET /api/v1/books/summary` がガイドのレスポンス例と一致する。
- `sort=price_asc` と `sort=price_desc` が価格順の結果を返す。
- `role=user` の登録リクエストが403になる。
- 入力不備の登録リクエストが400になる。
